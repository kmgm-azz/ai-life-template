# AI Life Template

やりたいことをGitHub Issuesへ集め、AIに整理・提案・更新してもらうための個人用テンプレートです。

自分で細かくタスク管理する代わりに、AIとの会話を入口にします。

- 「あとでこれをやりたい」→ AIがIssueとして整理
- 「今から何をすればいい？」→ AIが未完了Issueから候補を提案
- 「これを始めた／終わった」→ AIが進捗を更新

## 必要なもの

- GitHubアカウント
- リポジトリを読み書きできるAIエージェント

通常のAIチャットだけではGitHubを操作できません。GitHub連携や、ローカルのGitリポジトリを操作できる環境が必要です。

## 始め方

### 1. 自分用のリポジトリを作る

このページの **Use this template** → **Create a new repository** を選びます。

生活上の予定や個人的なメモを入れる場合は、リポジトリを **Private** にすることを推奨します。

### 2. Projectsの雛形をコピーする

[Projectsの雛形](https://github.com/users/kmgm-azz/projects/2)を開き、右上のメニューから **Make a copy** を選びます。

コピー後のProject名は `life` にすると、`AGENTS.md`の初期設定をそのまま使えます。別名にする場合は、`AGENTS.md`内のProject名も変更してください。

雛形の初期表示は、分類ごとにまとめたテーブルです。カンバン形式にしたい場合は、Projectの **View** メニューからレイアウトを **Board**、列の項目を **Status** に変更してください。必要に応じて **Slice by** で `分類` を選ぶと、分類を切り替えて確認できます。

### 3. AIに初期設定を頼む

作成したリポジトリをAIが操作できる状態にして、次のように頼みます。

> このリポジトリのAGENTS.mdを読んで、必要なラベルを確認・作成し、Project「life」と連携してください。

環境によってはGitHubへのログインや連携許可が別途必要です。

## 普段の使い方

特別な書式を覚える必要はありません。AIへ普通に話しかけます。

```text
富士山に登りたい。必要な準備をあとで整理したい。
```

```text
ブログの構想を思いついた。Issueにして。
```

```text
今から何をすればいい？
```

```text
競技プログラミングの教材を始めた。進捗を反映して。
```

AIは`AGENTS.md`のルールに従い、重複確認、締め切りの確認、分類、Projectへの追加などを行います。

詳しい会話例は[examples/conversations.md](examples/conversations.md)にあります。

## 仕組み

| 役割 | 使用するもの |
| --- | --- |
| AIへの運用マニュアル | `AGENTS.md` |
| 一つひとつのやること | GitHub Issues |
| 一覧・進捗確認 | GitHub Projects |
| 日付単位の記録 | `daily/` |
| 育てている構想 | `ideas/` |
| 繰り返し参照する情報 | `references/` |

## IssueとMarkdownの関係

Issueは相談や進捗管理の場所であり、自動的にMarkdownファイルへ変わるものではありません。

記事や資料をMarkdownとして残す場合は、次の流れにします。

1. Issueで構想や完了条件を整理する
2. 作業ブランチで`.md`ファイルを作る
3. Pull Requestを作り、Issueと関連付ける
4. 内容に満足したらPull Requestをマージする
5. Issueをクローズする

Pull Requestの本文へ `Closes #13` のように書くと、そのPull Requestがデフォルトブランチへマージされたとき、対応するIssueを自動でクローズできます。

noteやはてなブログなど外部サービスへ直接投稿する場合は、MarkdownのPull Requestは必須ではありません。Issueで執筆を管理し、公開後にURLを記録してIssueをクローズする運用でも問題ありません。

## 自分向けに変更する場所

まずは`AGENTS.md`の以下を変更すると、自分の使い方に合わせられます。

- Areaラベルの種類
- よく使うトピックラベル
- 「今日やること」の選定基準
- Issue作成時に確認してほしい情報
- Project名と項目

最初から完璧に設計する必要はありません。使いながら、AIへ「この運用を変えて」と頼んで更新していく前提です。

## セキュリティ

- APIキー、トークン、パスワードをIssueやファイルへ書かない
- 住所、連絡先、予約番号などの個人情報を不用意に保存しない
- 公開リポジトリでは生活予定や私的なIssueを扱わない
- AIが作成・更新した内容は、外部公開前に確認する
